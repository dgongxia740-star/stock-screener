# 日本株スイングトレード・スクリーニングツール

毎日決まった時刻に、設定ファイルで定義した条件を満たす日本株を機械的に抽出するローカルCLIツールです。

> ⚠️ **免責**
> 本ツールは**投資助言ツールではありません**。事前に定義したルールに合致する銘柄を
> 機械的に抽出して一覧表示するだけのものです。**売買判断はすべて利用者自身が行ってください。**
> 抽出結果は売買の推奨ではありません。

> ⚠️ **データについて**
> 本ツールは [yfinance](https://github.com/ranaroussi/yfinance)（Yahoo! Finance 非公式API）から
> データを取得します。価格は**おおむね15〜20分遅延**しており、計算は**日足（1日足）ベース**です。
> リアルタイム取引や場中の判断には適しません。終値ベースのスイングトレード用途を想定しています。

## セットアップ（初心者向け・コピペでOK）

### 0. 事前準備：Python 3.10以上が入っているか確認

ターミナル（macOSなら「ターミナル.app」、Windowsなら「PowerShell」）を開いて:

```bash
python3 --version
```

`Python 3.10.x` 以上が表示されればOKです（本ツールは 3.11.7 で動作確認済み）。
表示されない／古い場合は [python.org](https://www.python.org/downloads/) から最新版を入れてください。

### 1. このフォルダに移動

```bash
cd /path/to/stock-screener      # 実際の保存場所に置き換えてください
```

### 2. 仮想環境（venv）を作る

仮想環境とは「このツール専用の、隔離されたライブラリ置き場」です。
これを使うと、PC全体のPython環境を汚さずに済みます。**最初の1回だけ**実行します。

```bash
python3 -m venv .venv
```

実行すると `.venv` というフォルダが作られます（これが専用置き場です）。

### 3. 仮想環境を「有効化」する

ツールを使うたびに（ターミナルを開き直すたびに）実行します。

- **macOS / Linux:**
  ```bash
  source .venv/bin/activate
  ```
- **Windows (PowerShell):**
  ```powershell
  .venv\Scripts\Activate.ps1
  ```

有効化されると、プロンプトの先頭に `(.venv)` と表示されます。

### 4. 必要なライブラリをインストール

これも**最初の1回だけ**でOKです（`requirements.txt` のバージョンに固定されます）。

```bash
pip install -r requirements.txt
```

導入されるもの: `yfinance` / `pandas` / `PyYAML` / `rich`

## スクリーニング条件（すべて AND）

| # | 条件 | 内容 |
|---|------|------|
| 1 | トレンド | 25日移動平均（25MA）が**上向き**（`slope_lookback` 営業日前より高い） |
| 2 | 押し目 | **RSI(14)** が `min`〜`max` の範囲内（Wilder平滑） |
| 3 | 出来高 | 直近の出来高が**過去 `avg_window` 日平均出来高以上** |

各閾値は `config/rules.yaml` で調整できます（コード変更不要）。

## 出力項目（通過銘柄ごと）

- 銘柄コード / 名称、終値
- 25MA とその傾き（`現在の25MA − slope_lookback営業日前の25MA`）
- RSI(14)
- 出来高 / 20日平均比
- **推奨損切り価格** = `max(終値 × (1 − pct), 直近 low_lookback 日の安値)`
- **推奨株数** = `floor( (総資金 × リスク割合) ÷ (終値 − 損切り価格) )`
  - 終値 ≤ 損切り価格でリスク幅が取れない場合は **株数 0（建て見送り）** と表示します。

## マクロ・フィルタ（結果ヘッダに警告表示）

- `^N225`（日経平均）が 200日線の下 → **⚠相場全体が弱い：新規建て縮小推奨**
- `^VIX` が `vix_warn`(初期25)超 → **⚠VIX高：新規半減**
- `^VIX` が `vix_halt`(初期30)超 → **⚠VIX警戒：新規停止**

マクロ指標が取得できない場合も処理は止めず「判定不能」と表示します。

## 設定ファイル

すべての設定は `config/` 配下で完結します。

### `config/rules.yaml`
閾値・資金・損切り・マクロの各パラメータ。主な項目:

| 項目 | 初期値 | 説明 |
|------|--------|------|
| `risk.total_capital` | 1,000,000 | 総資金（円） |
| `risk.risk_ratio` | 0.01 | 1トレードの許容リスク割合（1%） |
| `stop_loss.pct` | 0.07 | 終値からの損切り下落率（7%） |
| `stop_loss.low_lookback` | 10 | 損切りに使う直近安値の日数 |
| `rsi.min` / `rsi.max` | 35 / 45 | RSIレンジ |
| `trend.ma_window` | 25 | 移動平均の期間 |
| `trend.slope_lookback` | 5 | 上向き判定の比較営業日数 |
| `volume.avg_window` | 20 | 平均出来高の期間 |

### `config/universe.txt`
対象銘柄リスト（1行1コード、yfinance形式 `7203.T`）。
`#` 以降はコメント。コード右側のコメントは銘柄名として表示に使われます。
初期はサンプルとして日経225の代表的な大型株10銘柄が入っています。
本番では日経225全銘柄などに差し替えてください。

## 堅牢性に関する設計

- **取得失敗で全体を止めない**: 1銘柄の取得に失敗してもスキップし、結果末尾に「取得失敗」として残します。
- **データ欠損・休場日**: 終値が欠損している行は除去し、指標計算に必要な行数が足りない銘柄は「データ不足でスキップ」とします。
- **リトライ**: 取得失敗時は指数バックオフで最大 `data.max_retries` 回リトライします。
- **キャッシュ**: 当日分のデータがあれば再取得せず、API負荷を抑えます。

# 紙トレード運用手順（実弾投入前の前向き検証）

バックテストの結論：**優位性は薄く（コスト後 約+0.08R/トレード）・上昇相場に偏り・下落局面では不安定**。
5年検証の好成績にはサバイバーシップ・バイアスと直近の上昇相場の綾が混ざっており、
バックテストだけでは「本物の優位性」と断言できません。そこで **実弾は入れず、まず紙トレード（ペーパートレード）で前向きに白黒つける**段階です。
紙トレードは現在の母集団に対するリアルタイム検証なので、サバイバーシップ・バイアスが原理的になく、過去最適化のしようもない真のアウトオブサンプル検証になります。

## 0. 開始前に確定するルール（途中で変えない）

紙トレードを始める前に、以下を「動かさない前提条件」として固定します。**検証期間中の途中変更は禁止**（変えると検証が無意味になります）。

| 項目 | ルール |
|---|---|
| **リスク** | **0.5%／トレード**（`rules.yaml` の `risk.risk_ratio: 0.005`。資金建てDDを実額で圧縮するため） |
| **位置づけ** | **「上昇相場の押し目ツール」と割り切る**。下落・もみ合い局面では稼げない前提。プラスを期待しない |
| **マクロフィルタ** | **継続必須**。日経平均が200日線の下にある日は新規エントリーを行わない（外すと下落局面で崩れる） |
| **同時保有** | 最大5銘柄。候補超過時は **RSIが低い順（押し目が深い順）** に5銘柄まで |
| **撤退基準①** | 前向き成績の期待値が **+0.08R/トレードを継続的に下回る**なら停止 |
| **撤退基準②** | 次の調整・下落局面で **累計 −10R を超えて沈む**なら停止（優位性は綾だったと判断） |

> 撤退基準のいずれかに触れたら、**実弾には進まずプロジェクトを終了**します。これは「損切りの損切り」＝戦略自体のストップです。

## 1. 毎日の流れ

エグジット条件は **バックテストで検証したルールに合わせます**（損切り −7%固定・利確 +15%・最大20営業日）。

### ① 引け後（その日の取引終了後）
```bash
cd stock-screener && source .venv/bin/activate
python screen.py
```
- ヘッダのマクロ警告を確認。**「⚠相場全体が弱い：新規建て縮小推奨」（日経が200日線の下）が出ている日は、新規エントリーを一切行わない**（マクロフィルタ＝撤退基準の前提）。
- 通過銘柄が出たら、保有中でない銘柄を **RSIが低い順に、空き枠（最大5）まで**「翌営業日の寄りで買う候補」として控える。

### ② 翌営業日の寄り付き（紙の上で約定）
- 寄り値（始値）を**約定価格**として記録シートに記入（これが `entry_price`）。
- 同時に各価格を確定：
  - **損切り** `stop_price = entry_price × 0.93`（−7%固定。※`screen.py` の表示する損切りは別式なので、紙トレードでは −7%固定を使う）
  - **利確** `target_price = entry_price × 1.15`（+15%）
  - **株数** `floor( 総資金 × 0.005 ÷ (entry_price − stop_price) )`
  - **時間切れ期限** = エントリー日を0日目として **20営業日後**

### ③ 保有中の管理（毎日、引け後にチェック）
次のいずれかに当てはまったら、**翌寄りではなくその条件の価格／タイミングで手仕舞い**を記録：
- **損切り**：日中安値が `stop_price` 以下 → `stop_price` で約定（寄りが割れていたら寄り値で。＝ギャップは想定より大きい損失）
- **利確**：日中高値が `target_price` 以上 → `target_price` で約定（寄りが上抜けていたら寄り値で）
- **時間切れ**：20営業日経過しても未決着 → その日の終値で手仕舞い
- 同日に損切り・利確の両方に触れた場合は **損切り優先**（保守的に記録）

## 2. 記録シート（[data/paper_trades_template.csv](stock-screener/data/paper_trades_template.csv) をコピーして使用）

1トレード＝1行。`pnl_R` は `(exit_price − entry_price) ÷ (entry_price − stop_price)` から手数料往復0.2%を引いた値で記録します（バックテストと同じR建て）。

| 列 | 内容 |
|---|---|
| `signal_date` | シグナルが出た日（引け後スクリーニング日） |
| `entry_date` | 翌営業日（紙の約定日） |
| `code` / `name` | 銘柄コード／名称 |
| `entry_price` | 寄りの約定価格 |
| `stop_price` | 損切り価格（entry×0.93） |
| `target_price` | 利確価格（entry×1.15） |
| `shares` | 株数（0.5%リスクで逆算） |
| `planned_risk_R` | 計画リスク（常に 1.0R） |
| `exit_date` / `exit_price` | 手仕舞い日／価格 |
| `exit_reason` | `stop` / `target` / `timeout` のいずれか |
| `pnl_R` | 損益（R建て、手数料控除後） |
| `regime` | その時の地合い（`up`＝日経200日線上 / `weak`＝下） |
| `notes` | 任意メモ（ギャップ約定など） |

定期的に `pnl_R` の平均（＝期待値R）・勝率・累計R・最大ドローダウンを集計し、撤退基準と照らし合わせます。

## 3. いつ実弾を検討するか（目安）

**最低でも次の調整・下落局面を1回はまたぐまで**続けてから判断します。具体的な目安：

- **期間：最低6ヶ月、できれば12ヶ月**
- **トレード数：最低30〜40件**（薄い優位性ほど統計的にはもっと多くの標本が要るため、少数では「方向性の確認」にとどまる点に留意）
- **必須条件：日経が200日線を割る「下落局面」を最低1回経験すること**

### なぜ「下落局面を1回はまたぐ」ことが必須か
この戦略の最大の弱点は、**優位性が上昇相場に偏っていること**でした。バックテストでも下落局面の期待値は標本によって符号が反転し（−0.29R 〜 +0.16R）、しかも5年版のプラスはサバイバーシップ・バイアスの疑いが濃厚でした。
つまり **「下落・もみ合い局面で本当はどう振る舞うか」は、前向きに一度経験するまで分からない**のが核心の未確認リスクです。上昇相場しか経験していない紙トレード結果で実弾に進むと、最も知りたいリスク（地合い悪化時の挙動）を確かめないまま本番投入することになります。**下落局面を1回くぐり抜けて、撤退基準②（−10R超）に触れずに期待値+0.08Rを維持できたなら**、初めて実弾を小さく検討する段階です。

## 4. 実弾検討の判定チェックリスト（紙トレード終了時）

- [ ] 期間6〜12ヶ月／トレード30件以上を満たした
- [ ] 下落局面を最低1回またいだ
- [ ] 期待値が +0.08R/トレードを継続的に上回っている
- [ ] 下落局面で累計 −10R を超えて沈まなかった
- [ ] マクロフィルタを一度も外さず運用した

**すべて満たす → 実弾を最小サイズで検討。1つでも欠ける → 優位性は綾と判断し、実弾には進まない。**
